## Protokol (HTTP vs HTTPS)
URL selalu diawali dengan protokol. 
- **HTTP**: Protokol standar untuk pengiriman data.
- **HTTPS**: Versi aman yang menggunakan enkripsi (SSL/TLS). Saat ini, HTTPS wajib digunakan untuk melindungi data user dan meningkatkan peringkat SEO.

## Domain dan Subdomain
Domain adalah alamat utama website Anda (misal: `kelasweb.id`). **Subdomain** adalah bagian tambahan di depan domain utama untuk memisahkan fungsi website (misal: `admin.kelasweb.id` untuk panel admin atau `blog.kelasweb.id` untuk artikel).

## Path (Halaman spesifik)
Path adalah bagian setelah nama domain yang menunjukkan lokasi halaman tertentu di server. Contohnya pada `google.com/search`, kata `/search` adalah path-nya. Path mempermudah kita dalam mengorganisir ribuan halaman dalam satu website.

## Query Parameters
Pernahkah Anda melihat URL yang sangat panjang dengan tanda tanya? Itulah **Query Parameters**. Contoh: `?q=belajar+coding`. Bagian ini digunakan untuk mengirimkan data tambahan ke server, seperti kata kunci pencarian atau filter kategori produk.
