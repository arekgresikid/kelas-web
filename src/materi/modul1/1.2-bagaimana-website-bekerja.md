## Konsep client-server
Website bekerja berdasarkan arsitektur **Client-Server**. 
- **Client**: Perangkat yang Anda gunakan (Laptop, HP) beserta aplikasinya (Browser). Client berperan sebagai peminta (requester).
- **Server**: Komputer bertenaga besar yang menyimpan file website (HTML, CSS, JS, Gambar) dan bertugas melayani permintaan (provider). Server biasanya terletak di gedung khusus yang disebut Data Center.

## Peran internet dan koneksi
Internet adalah "jalan raya" digital yang menghubungkan milyaran perangkat. Saat Anda mengakses website, data Anda dipecah menjadi bagian-bagian kecil yang disebut **paket**. Paket ini dikirim melalui kabel bawah laut, satelit, hingga router rumah Anda sampai tiba di tujuan dalam hitungan milidetik.

## HTTP request dan response
Komunikasi antara client dan server mengikuti protokol **HTTP (Hypertext Transfer Protocol)**.
1. **Request**: Browser mengirim "surat permintaan". Isinya meliputi metode (GET untuk ambil data, POST untuk kirim data), alamat tujuan, dan informasi browser.
2. **Response**: Server menjawab dengan "surat balasan" yang berisi data website yang diminta beserta status kodenya.

## Status code penting
Setiap jawaban dari server memiliki kode angka untuk memberi tahu browser apa yang terjadi:
- **2xx (Success)**: Semuanya berjalan lancar. Contoh: `200 OK`.
- **3xx (Redirection)**: Halaman yang Anda cari pindah tempat.
- **4xx (Client Error)**: Kesalahan ada pada permintaan Anda. Contoh: `404 Not Found` (salah ketik URL).
- **5xx (Server Error)**: Kesalahan ada pada sisi server. Contoh: `500 Internal Server Error` (server sedang mogok).
