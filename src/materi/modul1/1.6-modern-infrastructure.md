## 1. Evolusi Hosting ke Cloud
Dahulu, kita menyewa satu komputer (Server) untuk menaruh website. Sekarang, kita menggunakan **Cloud Computing** yang lebih fleksibel:

-   **Tradisional**: Jika server mati, website mati. Jika trafik naik, website lambat.
-   **Modern (Cloud)**: Website tersebar di ribuan server. Jika satu mati, yang lain membackup. Trafik naik? Sistem otomatis menambah kapasitas (Scaling).

---

## 2. Alur Distribusi Global (Edge Computing)
Untuk website yang sangat cepat, kita menggunakan teknologi **Edge**:

1.  **Origin Server**: Tempat kode utama Anda berada (misal: di Amerika).
2.  **CDN / Edge Nodes**: Salinan website Anda diletakkan di server-server kecil di seluruh dunia (termasuk Jakarta, Surabaya, dll).
3.  **User Experience**: Saat orang Indonesia membuka website Anda, data diambil dari server terdekat (Jakarta), bukan dari Amerika. Ini yang membuat website terasa instan.

---

## 3. Komponen Utama Infrastruktur
-   **Compute (Vercel/Cloudflare Workers)**: Tempat menjalankan logika kode tanpa perlu mengelola server (Serverless).
-   **Storage (S3/R2)**: Tempat menyimpan file gambar, video, dan aset statis.
-   **Database (Neon/PlanetScale)**: Tempat menyimpan data yang bisa diakses dari mana saja secara cepat.
-   **Security (WAF/SSL)**: Tameng yang melindungi website dari serangan hacker dan bot jahat.

---

## 4. Mengapa Memilih Cloudflare/Vercel?
Di KelasWeb, kita fokus pada platform ini karena:
- **Gratis untuk Pemula**: Anda bisa memulai tanpa biaya.
- **Auto-Deployment**: Cukup hubungkan ke GitHub, website otomatis update saat Anda simpan kode.
- **Kecepatan Dunia**: Website Anda langsung tersedia di ratusan lokasi di seluruh dunia secara otomatis.
