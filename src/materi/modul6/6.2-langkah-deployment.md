Setelah semua persiapan di materi sebelumnya selesai, sekarang saatnya kita melakukan "sihir" yang sesungguhnya: mengubah kode di laptop Anda menjadi link website asli yang bisa dibuka siapa saja.

## 1. Jembatan Utama: GitHub
Cara paling modern (dan paling disukai perusahaan teknologi) untuk deploy adalah melalui **GitHub**. 

*   Pastikan Anda sudah melakukan `git push` semua kode terbaru ke repository GitHub.
*   **Kenapa harus GitHub?** Karena jika Anda mengubah kode di laptop dan melakukan `push`, website Anda di internet akan **update otomatis** dalam hitungan detik. Ini disebut **CI/CD** (Continuous Integration/Continuous Deployment).

## 2. Proses Deploy 3-Klik (Contoh: Vercel)
Vercel adalah pilihan paling populer untuk aplikasi React/Next.js. Berikut langkah-langkahnya:

1.  **Login ke Vercel.com**: Gunakan akun GitHub Anda.
2.  **Klik "Add New" > "Project"**: Vercel akan menampilkan daftar repository GitHub Anda.
3.  **Klik "Import"**: Pilih project yang ingin Anda rilis.
4.  **Konfigurasi (Opsional)**: Vercel biasanya sudah tahu kalau Anda menggunakan Vite atau Next.js, jadi Anda tidak perlu mengubah settingan apa pun.
5.  **Klik "Deploy"**: Tunggu sekitar 1-2 menit, dan *Boom!* Website Anda sudah live dengan domain kustom dari Vercel (contoh: `project-anda.vercel.app`).

## 3. Memasukkan "Kunci Rahasia" (Environment Variables)
Ingat materi tentang file `.env`? Karena file tersebut tidak kita upload ke GitHub, kita harus memasukkannya secara manual ke dashboard Vercel:

*   Buka dashboard project Anda di Vercel.
*   Pilih menu **Settings** > **Environment Variables**.
*   Masukkan **Key** (contoh: `VITE_GEMINI_API_KEY`) dan **Value** (isi kuncinya).
*   **Penting**: Setelah memasukkan ini, Anda biasanya perlu melakukan **Redeploy** agar website mengenali kunci tersebut.

## 4. Jebakan Batman: Kenapa Build Error?
Banyak pemula stres karena website-nya "Error" saat deploy padahal di laptop lancar. Berikut adalah 3 alasan utamanya:

1.  **Masalah Huruf Besar/Kecil (Case Sensitivity)**: Di Windows, memanggil file `Header.tsx` atau `header.tsx` mungkin tidak masalah. Tapi di server Linux (Vercel/Netlify), **KEDUANYA BERBEDA**. Pastikan nama file di kode Anda persis sama dengan nama file aslinya.
2.  **Lupa Push File**: Kadang ada file baru yang belum Anda `git add` sehingga tidak terunggah ke GitHub.
3.  **Library Hilang**: Pastikan semua library yang Anda gunakan sudah terdaftar di `package.json`.

## 5. Fitur "Link Preview"
Setiap kali Anda membuat perubahan (misal di branch baru), Vercel akan membuatkan **link sementara**. Ini sangat berguna untuk:
*   Melihat hasil perubahan sebelum website utama diganti.
*   Meminta feedback dari teman atau klien tanpa merusak website asli yang sedang berjalan.

---

> [!IMPORTANT]
> **Jangan pernah deploy manual via drag-and-drop!** Selalu gunakan GitHub agar Anda punya riwayat perubahan kode dan website Anda selalu sinkron dengan apa yang ada di laptop.
