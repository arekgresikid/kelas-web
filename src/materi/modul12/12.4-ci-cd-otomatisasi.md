## 1. Apa itu CI/CD?
- **CI (Continuous Integration)**: Otomatisasi pengetesan kode setiap kali developer melakukan *push* ke GitHub.
- **CD (Continuous Deployment)**: Otomatisasi pengiriman kode yang sudah lolos tes ke server produksi.

---

## 2. Alur Kerja DevOps Modern
1.  **Coding**: Anda menulis fitur baru di laptop.
2.  **Push**: Anda mengirim kode ke GitHub.
3.  **Build & Test**: GitHub Actions (atau tool lain) otomatis mengecek apakah ada error.
4.  **Deploy**: Jika semua aman, website otomatis terupdate di internet.

---

## 3. Keuntungan Otomatisasi
- **Minim Human Error**: Tidak perlu lagi upload file manual lewat FTP yang berisiko salah hapus.
- **Lebih Cepat**: Fitur baru bisa sampai ke pengguna hanya dalam hitungan menit setelah selesai dikoding.
- **Kepercayaan Diri**: Anda tahu kode Anda aman karena sudah melewati tes otomatis.

---

## 4. Tool Populer
- **GitHub Actions**: Gratis untuk project open source dan sangat mudah diintegrasikan dengan GitHub.
- **Vercel/Netlify Build**: Sangat populer untuk project frontend karena hampir tidak butuh konfigurasi.

---

## 5. Kesimpulan
CI/CD adalah standar profesional. Seorang developer yang memahami alur ini akan jauh lebih dihargai di dunia kerja karena ia bisa menjamin stabilitas aplikasi.
