## 1. Kenapa Keamanan Itu Penting?
Website yang bagus bukan hanya yang cepat dan cantik, tapi juga yang **aman**. Satu celah kecil bisa membocorkan data ribuan pengguna Anda.

---

## 2. Ancaman Umum: XSS (Cross-Site Scripting)
XSS terjadi saat penjahat menyisipkan kode JavaScript jahat ke website Anda melalui kolom input (seperti komentar).
- **Efek**: Pencurian cookie atau pengambilalihan akun.
- **Pencegahan**: Jangan pernah percaya input dari user. Selalu bersihkan (*sanitize*) data sebelum ditampilkan di layar.

---

## 3. SQL Injection
Ancaman ini menyerang database Anda melalui query yang dimanipulasi.
- **Contoh**: Mengisi kolom password dengan `' OR 1=1 --` untuk masuk tanpa akun.
- **Pencegahan**: Gunakan **Prepared Statements** atau **ORM** (seperti Drizzle atau Prisma) yang sudah otomatis menangani masalah ini.

---

## 4. HTTPS & SSL
Pastikan website Anda selalu menggunakan **HTTPS**. Ini memastikan data yang dikirim antara browser dan server dienkripsi sehingga tidak bisa "diintip" di tengah jalan.

---

## 5. Tips Aman: Prinsip "Least Privilege"
Berikan akses secukupnya saja. Jika seorang user hanya butuh membaca artikel, jangan beri dia akses untuk menghapus user lain di database.
