## Apa itu Environment Variables (.env)?
Bayangkan Anda punya kunci brankas. Anda tidak akan menaruh kunci itu di depan pintu rumah, bukan? Begitu juga dengan kode. **Environment Variables** adalah cara kita menyimpan data sensitif (seperti password database atau API Key) di luar kode utama.

## Kenapa Harus Disembunyikan?
Jika Anda menaruh password langsung di dalam file `App.tsx`, maka saat Anda mengunggah kode ke GitHub, **semua orang di dunia bisa melihat password Anda**. Ini sangat berbahaya!

## Cara Kerjanya:
1. Kita membuat file bernama `.env`.
2. Kita masukkan data di sana, contoh: `DATABASE_PASSWORD=rahasia123`.
3. Kita beritahu Git untuk mengabaikan file ini (menggunakan file `.gitignore`).
4. Di dalam kode, kita panggil menggunakan `process.env` atau `import.meta.env`.

## Hal Teknis Penting Lainnya:

### 1. HTTPS (SSL)
Selalu pastikan website Anda memiliki icon gembok hijau di browser. Ini memastikan data yang dikirim user (seperti password) dienkripsi dan tidak bisa diintip di tengah jalan.

### 2. CORS (Cross-Origin Resource Sharing)
Ini adalah sistem keamanan yang membatasi siapa saja yang boleh mengambil data dari API Anda. Ini mencegah website asing mencuri data dari server Anda.

> [!CAUTION]
> JANGAN PERNAH mengunggah file `.env` Anda ke GitHub atau layanan publik lainnya. Keamanan website Anda bergantung pada kerahasiaan file ini! 🔐
