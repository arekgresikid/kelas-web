## Kenapa Butuh Database?
Selama ini kita belajar membuat tampilan, tapi data yang kita masukkan akan hilang saat halaman di-refresh. Untuk menyimpan data selamanya (seperti data pengguna atau komentar), kita butuh **Database**.

## Jenis Database Modern
Untuk pemula, lupakan cara lama yang harus menginstal server database yang rumit. Gunakan layanan **Backend-as-a-Service (BaaS)**:

1. **Supabase / Firebase**: Database instan yang sudah termasuk sistem login dan penyimpanan file.
2. **Cloudflare D1**: Database SQL yang sangat cepat dan murah.
3. **Prisma**: Alat (ORM) yang memudahkan kita menulis perintah database menggunakan bahasa JavaScript biasa.

## Analogi Sederhana
Bayangkan website Anda adalah sebuah **Restoran**:
- **Frontend (HTML/CSS)**: Meja makan dan dekorasi yang dilihat pelanggan.
- **Backend (Logic)**: Pelayan yang mencatat pesanan.
- **Database**: Gudang bahan makanan di belakang. Tanpa gudang, pelayan tidak punya apa-apa untuk disajikan.

## Cara Kerja CRUD
Hampir semua aplikasi web melakukan 4 hal dasar ini terhadap data:
- **C (Create)**: Menambah data baru (misal: Daftar akun).
- **R (Read)**: Menampilkan data (misal: Melihat feed Instagram).
- **U (Update)**: Mengubah data (misal: Edit profil).
- **D (Delete)**: Menghapus data (misal: Hapus postingan).
