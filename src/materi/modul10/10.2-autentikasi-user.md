## Apa itu Autentikasi?
Autentikasi adalah proses membuktikan bahwa "Anda adalah Anda". Biasanya dilakukan dengan Email & Password, atau melalui **Social Login** (seperti Login with Google).

## Cara Kerja Sistem Login
Bukannya menyimpan password secara polos (yang sangat berbahaya), sistem modern bekerja seperti ini:
1. **User Login**: User memasukkan password.
2. **Hashing**: Password diacak menjadi kode rumit (Hash) yang tidak bisa dibaca manusia.
3. **Token (JWT)**: Setelah login berhasil, website memberikan "Kartu Akses" (Token) digital ke browser user.
4. **Access**: Selama kartu ini ada, user bisa mengakses halaman rahasia (Dashboard).

## Alat Populer untuk Pemula
Anda tidak perlu membangun sistem keamanan dari nol (sangat berisiko!). Gunakan layanan yang sudah teruji:
- **Clerk**: Sangat mudah dipasang di Next.js (Hanya butuh 5 menit).
- **Supabase Auth**: Gratis dan terintegrasi langsung dengan database.
- **NextAuth.js**: Standar industri untuk framework Next.js.

## Kenapa Harus Social Login?
Tahukah Anda bahwa user 80% lebih cenderung mendaftar jika ada tombol **"Login with Google"**? Ini karena mereka tidak perlu repot membuat password baru. Platform **KelasWeb** ini pun menggunakan Google Login untuk kenyamanan Anda.

> [!CAUTION]
> Jangan pernah mencoba menyimpan password user dalam bentuk teks biasa (Plain Text) di database. Jika database Anda bocor, reputasi Anda akan hancur seketika!
