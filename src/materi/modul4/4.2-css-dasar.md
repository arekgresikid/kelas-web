Jika HTML adalah tubuh, maka **CSS (Cascading Style Sheets)** adalah pakaian, riasan, dan cara tubuh tersebut berdiri di ruang tamu.

## 1. Kenapa Namanya "Cascading"?
Kata *Cascading* (Air Terjun) berarti gaya yang Anda tulis akan mengalir dari atas ke bawah.
*   **Prioritas**: Jika Anda memberi warna merah pada seluruh `<body>`, maka semua teks akan menjadi merah, kecuali Anda secara spesifik memberi warna biru pada tag `<p>`.
*   **Inheritance (Pewarisan)**: Elemen anak akan "mewarisi" sifat dari induknya.
    *   **Analogi Pohon Keluarga**: Jika Anda memberi perintah "Semua keluarga ini memakai baju seragam biru" pada kakek (tag `<body>`), maka anak dan cucu (tag `<div>`, `<span>`) akan otomatis ikut berbaju biru, kecuali ada cucu nakal yang secara spesifik minta baju merah.

## 2. Anatomi Box Model (Fondasi Terpenting)
Setiap elemen di web adalah sebuah **Kotak**. Jika Anda tidak paham ini, Anda akan sering stres saat mengatur jarak.
*   **Content**: Inti dari kotak (teks/gambar).
*   **Padding**: Ruang napas *di dalam* kotak (jarak antara teks dan bingkai).
*   **Border**: Bingkai atau garis tepi kotak.
*   **Margin**: Ruang napas *di luar* kotak (jarak antar kotak/elemen).

> **Analogi**: Jika Anda mengenakan jaket, **Padding** adalah ketebalan busa di dalam jaket, **Border** adalah kain luar jaket, dan **Margin** adalah jarak aman Anda dengan orang lain agar tidak bertabrakan.

## 3. Revolusi Tata Letak Modern
Lupakan cara lama yang rumit. Gunakan dua "Senjata Utama" ini:

### A. Flexbox (Ahli Perataan)
Sangat powerfull untuk mengatur elemen dalam satu baris (Horizontal) atau satu kolom (Vertikal).
*   **Gunakan untuk**: Navbar, tombol, atau menyelaraskan konten di tengah kotak secara instan.

### B. CSS Grid (Ahli Struktur)
Sangat hebat untuk membuat struktur halaman yang kompleks secara dua dimensi (Baris dan Kolom sekaligus).
*   **Gunakan untuk**: Layout majalah, galeri foto bento, atau kerangka utama halaman web.

## 4. Filosofi Unit: PX vs REM
Jangan gunakan `px` (pixel) untuk ukuran font di tahun 2026!
*   **PX**: Bersifat kaku. Jika pengguna memperbesar font di browser mereka, website Anda tetap kecil.
*   **REM**: Bersifat relatif. Ukuran font akan menyesuaikan dengan pengaturan sistem pengguna. Ini adalah standar **Aksesibilitas** agar website Anda nyaman dibaca oleh siapa saja, termasuk lansia.

## 5. Mindset "Mobile-First"
Developer amatir membuat web untuk monitor besar lalu pusing memperbaikinya untuk HP. **Developer Pro melakukan sebaliknya.**
1.  **Desain untuk HP Dulu**: Buat tampilan sesederhana mungkin untuk layar kecil.
2.  **Tambahkan Kompleksitas**: Saat layar makin lebar (Tablet/Laptop), barulah kita menambah kolom atau fitur tambahan.
*   **Kenapa?** Lebih dari 70% orang membuka web dari HP. Website yang bagus di HP biasanya otomatis bagus di monitor besar, tapi tidak sebaliknya.

## 6. Menuju Utility-First (Tailwind CSS)
Di industri modern, kita sering merasa menulis CSS secara manual sangat lambat dan menghasilkan file yang berantakan. Itulah sebabnya muncul **Tailwind CSS**.
*   Alih-alih menulis: `font-size: 1.5rem; color: blue;`, 
*   Kita cukup menulis class: `text-2xl text-blue-500`.
*   *Catatan: Kita akan menggunakan teknik ini di hampir seluruh project di KelasWeb!*
