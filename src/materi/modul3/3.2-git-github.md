## Apa itu Git?
Git adalah sistem kontrol versi (*version control system*) yang mencatat setiap perubahan pada file Anda. Bayangkan Git sebagai "mesin waktu" untuk kode Anda. Jika Anda membuat kesalahan, Anda bisa kembali ke versi sebelumnya dengan mudah. Git bekerja secara lokal di komputer Anda.

## Apa itu GitHub?
GitHub adalah platform berbasis cloud untuk menyimpan proyek Git Anda secara online. Jika Git adalah alatnya, GitHub adalah tempat penyimpanannya. 
**Peran GitHub:**
1. **Penyimpanan (Hosting)**: Menyimpan kode Anda agar aman dan bisa diakses dari mana saja.
2. **Kolaborasi**: Memungkinkan banyak orang bekerja pada proyek yang sama tanpa saling menimpa kode.
3. **Portofolio**: Menampilkan karya Anda kepada dunia atau calon pemberi kerja.

## Konsep Repository dan Commit
- **Repository (Repo)**: Folder proyek yang diawasi oleh Git.
- **Commit**: Sebuah "snapshot" atau rekaman dari perubahan yang Anda lakukan. Setiap commit harus memiliki pesan yang menjelaskan apa yang Anda ubah.
- **Push**: Mengirim commit lokal Anda ke GitHub.
- **Pull**: Mengambil perubahan terbaru dari GitHub ke komputer lokal Anda.

## Kolaborasi dengan Pull Request
Pull Request (PR) adalah cara Anda menawarkan perubahan kode kepada pemilik proyek. Sebelum kode digabungkan (*merged*), orang lain bisa memberikan komentar atau saran perbaikan. Ini adalah jantung dari pengembangan perangkat lunak modern.
