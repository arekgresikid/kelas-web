## Mengapa Butuh Google Cloud Console?

Untuk menggunakan fitur "Login with Google", website kamu harus terdaftar secara resmi di sistem Google. Google perlu tahu siapa yang meminta data profil user dan apakah website tersebut aman.

## Langkah-Langkah Membuat Credential

### 1. Buat Project Baru
1. Buka [Google Cloud Console](https://console.cloud.google.com/).
2. Login dengan akun Google kamu.
3. Klik dropdown project di pojok kiri atas dan klik **New Project**.
4. Beri nama (misal: `KelasWeb-Auth`) dan klik **Create**.

### 2. Konfigurasi OAuth Consent Screen
Sebelum buat kunci, kamu harus atur tampilan izinnya:
1. Cari menu **APIs & Services** -> **OAuth consent screen**.
2. Pilih User Type: **External** (agar bisa dipakai siapa saja) dan klik **Create**.
3. Isi data wajib:
   - App name: `KelasWeb`
   - User support email: Email kamu.
   - Developer contact info: Email kamu.
4. Klik **Save and Continue** sampai selesai (untuk belajar, lewati bagian Scopes).

### 3. Membuat OAuth Client ID
Sekarang saatnya mendapatkan "Kunci" (Client ID):
1. Buka menu **APIs & Services** -> **Credentials**.
2. Klik **+ Create Credentials** -> **OAuth client ID**.
3. Pilih Application type: **Web application**.
4. Beri nama: `KelasWeb-Local`.
5. **Authorized JavaScript origins**:
   - Tambahkan: `http://localhost:5173` (ini URL default Vite).
6. **Authorized redirect URIs**:
   - Tambahkan: `http://localhost:5173`.
7. Klik **Create**.

## Mengambil Client ID

Setelah klik Create, akan muncul pop-up berisi **Your Client ID**. 
1. Copy Client ID tersebut.
2. Buka file `.env` di folder proyek kamu.
3. Masukkan ke variabel:
   ```env
   VITE_GOOGLE_CLIENT_ID=isi_dengan_client_id_kamu
   ```

> [!WARNING]
> Jangan pernah membagikan **Client Secret** Anda kepada siapapun atau meng-upload-nya ke GitHub. Untuk aplikasi frontend, kita hanya membutuhkan **Client ID**.

## Langkah Selanjutnya
Setelah Client ID terpasang, fitur login di website kamu akan otomatis mengenali project Google kamu dan siap digunakan!
