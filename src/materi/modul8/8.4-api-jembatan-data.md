## 1. Apa itu API?
**API (Application Programming Interface)** adalah perantara yang memungkinkan dua aplikasi untuk saling berbicara. Jika website Anda ingin mengambil data dari database atau layanan lain (seperti Google Maps atau Cuaca), Anda menggunakan API.

> **Analogi**: API seperti **Pelayan Restoran**. Anda (Client) tidak langsung ke dapur (Database). Anda bicara ke pelayan (API), pelayan menyampaikan ke dapur, dan pelayan membawa makanan (Data) kembali ke Anda.

---

## 2. Cara Kerja REST API
Format API yang paling populer saat ini adalah **REST (Representational State Transfer)**. Ia menggunakan protokol HTTP dengan perintah-perintah utama:
-   **GET**: Mengambil data (misal: melihat daftar produk).
-   **POST**: Mengirim data baru (misal: mendaftar akun baru).
-   **PUT/PATCH**: Memperbarui data (misal: ganti foto profil).
-   **DELETE**: Menghapus data.

---

## 3. JSON: Bahasa Universal API
Saat dua aplikasi bicara lewat API, mereka biasanya menggunakan format **JSON (JavaScript Object Notation)**. Bentuknya sangat sederhana dan mudah dibaca:

```json
{
  "nama": "Budi",
  "pekerjaan": "Developer",
  "keahlian": ["HTML", "CSS", "React"]
}
```

---

## 4. Endpoint: Alamat API
Sama seperti website punya URL, API punya **Endpoint**. 
Contoh:
- `https://api.toko.com/products` (Untuk ambil semua produk)
- `https://api.toko.com/products/1` (Untuk ambil produk nomor 1)

---

## 5. Fetch API (Cara Memanggil di Kode)
Di dunia JavaScript modern, kita menggunakan fungsi `fetch()` untuk memanggil API.
```javascript
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data));
```

---

## 6. Mengapa API Sangat Penting?
1.  **Integrasi**: Menghubungkan website Anda dengan layanan pihak ketiga (Login Google, Bayar via Midtrans).
2.  **Keamanan**: Database Anda tetap aman di belakang (Server), tidak terekspos langsung ke browser.
3.  **Modular**: Anda bisa membangun Frontend dan Backend secara terpisah.
