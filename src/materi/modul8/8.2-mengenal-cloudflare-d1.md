## Apa itu Cloudflare D1?

**Cloudflare D1** adalah database SQL pertama dari Cloudflare yang dibangun di atas SQLite. Berbeda dengan database tradisional, D1 berjalan di jaringan "Edge" Cloudflare, yang artinya data kamu berada sangat dekat dengan pengguna di seluruh dunia.

Keunggulan D1 untuk proyek ini:
- **Gratis**: Cloudflare memiliki tier gratis yang sangat murah hati.
- **Serverless**: Kamu tidak perlu mengatur server, biarkan Cloudflare yang melakukannya.
- **Integrasi**: Sangat mudah dihubungkan jika kamu nantinya men-deploy website ini ke Cloudflare Pages.

## Cara Membuat Database D1 Pertama

1. Pastikan kamu sudah punya akun [Cloudflare](https://dash.cloudflare.com/).
2. Di dashboard, cari menu **Workers & Pages** -> **D1**.
3. Klik **Create database** dan beri nama, misalnya `db-kelasweb`.
4. Database kamu sekarang sudah siap!

## Membuat Tabel via Dashboard

D1 memungkinkan kamu membuat tabel menggunakan perintah SQL langsung di dashboard:
1. Klik database yang baru kamu buat.
2. Cari tab **Console**.
3. Kamu bisa menyalin isi dari file `schema.sql` yang ada di folder proyek kamu, atau masukkan perintah ini:
   ```sql
   CREATE TABLE authorized_users (
     id INTEGER PRIMARY KEY AUTOINCREMENT,
     email TEXT UNIQUE NOT NULL,
     name TEXT,
     created_at DATETIME DEFAULT CURRENT_TIMESTAMP
   );
   ```
4. Klik **Execute**.

## Menyiapkan Daftar Email Terdaftar

Agar login berfungsi, kamu harus memasukkan email ke dalam tabel tersebut:
```sql
INSERT INTO authorized_users (email, name) VALUES ('email-kamu@gmail.com', 'Nama Kamu');
```

> [!IMPORTANT]
> Karena D1 berjalan di sisi server Cloudflare, kita tidak bisa memanggilnya langsung dari browser (Frontend) demi keamanan. Kita butuh "Jembatan" berupa **Cloudflare Workers** atau **Pages Functions**.

## Langkah Selanjutnya
Kita akan membahas bagaimana membuat API sederhana menggunakan Cloudflare Workers untuk mengecek apakah email yang login ada di dalam database D1 ini!
