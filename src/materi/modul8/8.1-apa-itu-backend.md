## Dunia di Balik Layar

Jika Frontend adalah **ruang tamu** yang indah, maka Backend adalah **dapur dan gudang**. Pengunjung tidak melihat apa yang terjadi di dapur, tapi mereka merasakan hasilnya: makanan yang lezat (data yang benar).

Secara sederhana, Backend terdiri dari tiga komponen utama:
1. **Server**: Komputer yang selalu menyala untuk melayani permintaan.
2. **Database**: Tempat penyimpanan data yang terstruktur.
3. **API (Application Programming Interface)**: Pelayan yang mengantarkan pesan antara Frontend dan Backend.

## Mengapa Kita Butuh Backend?

Coba bayangkan jika Facebook tidak punya Backend:
- Kamu bisa mengedit profil, tapi begitu halaman di-refresh, semua perubahan hilang.
- Kamu tidak bisa login, karena tidak ada tempat untuk menyimpan password.
- Kamu tidak bisa melihat postingan teman, karena tidak ada data yang dibagikan secara global.

> [!TIP]
> Di era modern (2026), kita tidak harus membangun server sendiri dari nol. Kita bisa menggunakan layanan **Edge Database** seperti Cloudflare D1 yang sangat cepat dan terintegrasi.

## Bagaimana Mereka Bekerja Sama?

Alurnya seperti ini:
1. **Request**: Kamu menekan tombol "Like" (Frontend).
2. **API**: Mengirim pesan ke Server: "User A ingin me-like postingan B".
3. **Server**: Memvalidasi permintaan (apakah User A sudah login?).
4. **Database**: Mengupdate angka like di tabel database.
5. **Response**: API memberi tahu Frontend: "Berhasil!".
6. **Update**: Frontend menampilkan ikon jempol berwarna biru.

## Langkah Selanjutnya
Kita akan mencoba menggunakan **Cloudflare D1**, database SQL modern yang berjalan di jaringan Edge untuk performa maksimal.
