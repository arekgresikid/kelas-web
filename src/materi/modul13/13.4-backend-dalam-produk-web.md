## Apa itu backend?
Backend adalah bagian aplikasi yang berjalan di server atau lingkungan komputasi di luar browser pengguna. Jika frontend adalah antarmuka yang dilihat pengguna, backend adalah mesin yang memproses logika, menyimpan data, memeriksa izin, dan menghubungkan aplikasi dengan layanan lain. Backend dapat dibuat dengan Node.js, Python, Go, PHP, Ruby, Java, atau platform serverless seperti Cloudflare Workers, Vercel Functions, dan AWS Lambda.

Pengguna biasanya tidak melihat backend secara langsung, tetapi mereka merasakan dampaknya. Login yang aman, data progres yang tersimpan, pembayaran yang diproses, email yang terkirim, pencarian yang cepat, dan dashboard admin yang akurat semuanya bergantung pada backend yang baik.

## Tanggung jawab utama backend
Tanggung jawab pertama adalah mengelola data. Backend berkomunikasi dengan database untuk membuat, membaca, memperbarui, dan menghapus data. Database bisa relasional seperti PostgreSQL, MySQL, SQLite, atau Cloudflare D1; bisa juga NoSQL seperti MongoDB atau Firestore. Backend memastikan data tersimpan dengan struktur yang benar dan dapat diambil sesuai kebutuhan.

Tanggung jawab kedua adalah menjalankan aturan bisnis. Aturan bisnis adalah logika yang menentukan bagaimana produk bekerja. Contohnya: user hanya boleh mengakses progres miliknya sendiri, kupon diskon hanya berlaku sekali, sertifikat hanya muncul setelah semua materi selesai, atau admin dapat melihat daftar user. Aturan seperti ini sebaiknya tidak hanya ditempatkan di frontend karena frontend bisa dimodifikasi oleh pengguna.

Tanggung jawab ketiga adalah keamanan. Backend memvalidasi input, memeriksa autentikasi, mengatur otorisasi, melindungi secret, membatasi request berlebihan, dan mencegah serangan umum seperti SQL injection atau akses data tanpa izin. Frontend boleh membantu validasi, tetapi backend tetap menjadi penjaga utama integritas data.

## API sebagai kontrak frontend dan backend
Frontend dan backend biasanya berkomunikasi lewat API. API menentukan endpoint, method, request body, response body, status code, dan format error. Kontrak API yang jelas mengurangi kebingungan. Misalnya endpoint `GET /api/progress` mengembalikan daftar progress user, sedangkan `POST /api/progress` menyimpan perubahan progress.

Format error juga penting. Jika validasi form gagal, backend sebaiknya mengirim informasi field mana yang bermasalah. Jika user belum login, backend mengirim status 401. Jika user tidak punya izin, status 403. Jika data tidak ditemukan, status 404. Dengan status yang tepat, frontend dapat menampilkan pesan yang sesuai.

## Autentikasi dan otorisasi
Autentikasi menjawab pertanyaan “siapa pengguna ini?” Contohnya login dengan email/password, OAuth GitHub, OAuth Google, magic link, atau session cookie. Setelah pengguna dikenali, aplikasi membuat session atau token agar request berikutnya dapat dihubungkan dengan user tersebut.

Otorisasi menjawab pertanyaan “apa yang boleh dilakukan pengguna ini?” User biasa mungkin hanya boleh melihat datanya sendiri. Admin boleh mengelola banyak user. Mentor boleh mengakses kelas tertentu. Banyak bug keamanan terjadi karena aplikasi sudah memeriksa login tetapi belum memeriksa izin secara detail. Backend harus selalu memeriksa otorisasi pada aksi sensitif.

## Validasi input dan sanitasi
Semua input dari pengguna harus dianggap tidak tepercaya. Form bisa dimanipulasi, request bisa dikirim manual, dan JavaScript frontend bisa diubah. Backend harus memvalidasi tipe data, panjang teks, format email, nilai angka, dan relasi data. Jika aplikasi menerima konten HTML atau markdown, backend juga perlu mempertimbangkan sanitasi untuk mencegah XSS.

Validasi yang baik juga membantu UX. Daripada mengembalikan error umum, backend dapat mengirim pesan spesifik seperti “Email sudah terdaftar” atau “Password minimal 8 karakter.” Frontend kemudian menampilkan pesan tersebut di tempat yang tepat.

## Database dan struktur data
Desain database memengaruhi kemampuan aplikasi berkembang. Tabel atau collection perlu mencerminkan kebutuhan produk. Untuk aplikasi belajar, mungkin ada tabel users, modules, lessons, progress, certificates, dan roles. Relasi data harus jelas. Progress sebaiknya terhubung dengan user dan lesson. Certificate sebaiknya terhubung dengan user dan tanggal penerbitan.

Selain struktur, pikirkan indeks dan query. Data kecil mungkin terasa cepat tanpa optimasi, tetapi ketika user bertambah, query yang buruk bisa memperlambat aplikasi. Backend developer perlu memahami pola akses data: data apa yang sering dicari, difilter, diurutkan, atau digabungkan.

## Integrasi layanan eksternal
Backend sering menghubungkan aplikasi dengan layanan eksternal seperti payment gateway, email provider, storage, analytics, AI API, atau webhook. Integrasi ini biasanya membutuhkan secret key yang tidak boleh berada di frontend. Backend menjadi tempat aman untuk menyimpan dan menggunakan secret tersebut.

Webhook adalah contoh penting. Payment gateway dapat mengirim webhook ketika pembayaran berhasil. Backend menerima webhook, memverifikasi tanda tangan, lalu memperbarui status pesanan. Jika proses ini hanya dilakukan di frontend, pengguna bisa memalsukan status pembayaran. Karena itu, proses sensitif harus berada di backend.

## Observability dan maintenance
Backend yang baik tidak hanya berjalan, tetapi juga bisa dipantau. Logging, error tracking, metrics, dan alert membantu developer mengetahui masalah sebelum banyak pengguna terdampak. Log sebaiknya cukup informatif untuk debugging, tetapi jangan menyimpan data sensitif seperti password atau token.

Maintenance backend mencakup migrasi database, backup, rotasi secret, update dependency, dan pengujian endpoint. Semakin penting aplikasi, semakin penting proses operasionalnya. Backend adalah fondasi kepercayaan: jika data hilang atau akses bocor, tampilan frontend sebagus apa pun tidak cukup menyelamatkan produk.

## Latihan praktis
Rancang backend sederhana untuk aplikasi kursus. Buat daftar data yang perlu disimpan: user, materi, progress, dan sertifikat. Tentukan endpoint yang dibutuhkan: login, ambil daftar materi, simpan progress, dan ambil sertifikat. Untuk setiap endpoint, tulis siapa yang boleh mengakses, data apa yang dikirim, respons sukses, dan respons error. Latihan ini membantu Anda memahami backend sebagai sistem logika dan keamanan, bukan hanya tempat menyimpan data.
