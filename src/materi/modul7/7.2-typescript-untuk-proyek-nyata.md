## Mengapa TypeScript penting setelah memahami JavaScript?
TypeScript adalah lapisan di atas JavaScript yang menambahkan sistem tipe statis. Artinya, sebelum aplikasi dijalankan di browser atau server, editor dan compiler dapat membantu menemukan kesalahan seperti salah nama properti, tipe data yang tidak sesuai, atau fungsi yang dipanggil dengan argumen keliru. Bagi pemula, TypeScript kadang terlihat menambah kerumitan karena kita harus menulis `string`, `number`, `boolean`, `interface`, atau `type`. Namun dalam proyek nyata, terutama ketika file semakin banyak dan fitur saling terhubung, TypeScript justru membuat proses pengembangan lebih tenang.

Bayangkan Anda membuat aplikasi kelas online. Ada data pengguna, progres belajar, daftar modul, pembayaran, sertifikat, dan halaman admin. Tanpa tipe yang jelas, satu perubahan kecil pada struktur data dapat merusak banyak halaman tanpa terlihat sampai pengguna menemukan error. Dengan TypeScript, bentuk data dapat didefinisikan sejak awal. Jika properti `title` pada materi berubah menjadi `name`, editor akan langsung menandai bagian kode yang masih memakai `title`. Ini membantu tim bergerak cepat tanpa mengorbankan keandalan.

## Tipe dasar dan pola berpikirnya
Tipe dasar TypeScript meliputi `string`, `number`, `boolean`, `null`, `undefined`, array, object, dan union. Yang paling penting bukan menghafal semua sintaks, tetapi memahami pertanyaan yang harus dijawab saat menulis kode: data ini bentuknya apa, boleh kosong atau tidak, berasal dari mana, dan akan dipakai oleh siapa? Misalnya, judul materi hampir selalu berupa teks, urutan materi berupa angka, sedangkan status selesai berupa boolean. Jika data datang dari API, kita harus lebih hati-hati karena data eksternal bisa berubah atau tidak lengkap.

Contoh sederhana:

```ts
type Materi = {
  id: string;
  title: string;
  order: number;
  isCompleted: boolean;
};

const materiPertama: Materi = {
  id: 'html-dasar',
  title: 'HTML Dasar',
  order: 1,
  isCompleted: false,
};
```

Kode di atas memberi kontrak yang jelas. Setiap objek `Materi` harus punya empat properti tersebut dengan tipe yang sesuai. Jika `order` tidak sengaja diisi teks seperti `'satu'`, TypeScript akan memberi peringatan. Ini tampak sederhana, tetapi efeknya besar ketika objek yang sama dipakai di banyak komponen.

## Interface, type alias, dan kapan memakainya
Dua cara populer mendefinisikan bentuk data adalah `interface` dan `type`. Keduanya sering bisa dipakai bergantian. `interface` terasa natural untuk menggambarkan bentuk object, terutama props komponen React atau respons API. `type` lebih fleksibel untuk union, kombinasi tipe, atau alias sederhana. Dalam proyek tim, yang terpenting adalah konsistensi. Jika tim memilih `interface` untuk data model dan `type` untuk union state, ikuti pola itu agar kode mudah dibaca.

Contoh union yang sering muncul pada UI:

```ts
type LoadingState = 'idle' | 'loading' | 'success' | 'error';
```

Dengan union seperti ini, komponen tidak bisa sembarangan mengisi status dengan teks acak. Status hanya boleh salah satu dari empat pilihan. Pola ini sangat membantu ketika membuat halaman yang mengambil data dari server. Daripada memiliki banyak boolean seperti `isLoading`, `isError`, dan `isSuccess` yang bisa saling bertabrakan, satu state eksplisit sering lebih mudah dikelola.

## TypeScript di React
Dalam React, TypeScript paling sering digunakan untuk props, state, event, dan data hasil fetch. Props adalah kontrak antara komponen induk dan komponen anak. Jika komponen `ModuleCard` membutuhkan `title`, `description`, dan `progress`, tuliskan tipe props agar pengguna komponen tidak salah mengirim data.

```tsx
type ModuleCardProps = {
  title: string;
  description: string;
  progress: number;
};

function ModuleCard({ title, description, progress }: ModuleCardProps) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{description}</p>
      <span>{progress}% selesai</span>
    </article>
  );
}
```

Ketika props sudah jelas, komponen menjadi lebih mudah digunakan ulang. Editor akan menampilkan autocomplete, dokumentasi singkat, dan error jika ada props wajib yang belum dikirim. Ini membuat proses belajar React lebih terarah karena Anda tidak hanya melihat tampilan, tetapi juga memahami struktur data yang menggerakkan tampilan.

## Menangani data eksternal dengan aman
Satu kesalahan umum adalah menganggap TypeScript otomatis menjamin semua data benar. TypeScript bekerja pada waktu pengembangan, sedangkan data API datang saat runtime. Jika server mengirim data rusak, TypeScript tidak otomatis menghentikannya. Karena itu, untuk data penting, gunakan validasi runtime atau minimal lakukan pemeriksaan sederhana sebelum menampilkan data. Misalnya, cek apakah respons memiliki array, apakah properti wajib tersedia, dan apakah nilai angka berada dalam rentang yang masuk akal.

Pada proyek kecil, validasi manual sering cukup. Pada proyek yang lebih serius, gunakan library seperti Zod, Valibot, atau Yup untuk membuat schema validasi. Schema tersebut dapat memastikan data login, form pendaftaran, pengaturan profil, atau payload pembayaran sesuai aturan. Dengan menggabungkan TypeScript dan validasi runtime, aplikasi menjadi lebih tahan terhadap bug dari input pengguna maupun perubahan API.

## Strategi migrasi dari JavaScript
Jika Anda sudah memiliki proyek JavaScript, jangan ubah semuanya sekaligus. Mulailah dari file yang paling sering berubah atau paling rawan bug, seperti model data, komponen form, dan fungsi pemanggil API. Aktifkan TypeScript secara bertahap, perbaiki error yang paling jelas, lalu tingkatkan aturan compiler setelah tim nyaman. Hindari menutup semua error dengan `any` karena itu menghilangkan manfaat TypeScript. Jika memang belum tahu tipe pastinya, gunakan `unknown` lalu lakukan pengecekan sebelum dipakai.

Target yang realistis untuk pemula adalah memahami tipe object, array, union, props React, dan tipe return function. Setelah itu baru pelajari generic, utility type, discriminated union, dan konfigurasi `tsconfig`. TypeScript bukan tujuan akhir, melainkan alat untuk membuat aplikasi lebih mudah dirawat.

## Latihan praktis
Ambil satu komponen di proyek Anda, lalu tulis tipe props-nya. Setelah itu, definisikan tipe untuk data utama aplikasi, misalnya `User`, `Product`, `Post`, atau `Materi`. Coba ubah salah satu properti dan perhatikan bagaimana editor membantu menemukan bagian yang terdampak. Latihan sederhana ini akan menunjukkan nilai TypeScript secara langsung: bukan hanya membuat kode terlihat profesional, tetapi juga membantu Anda berpikir lebih sistematis tentang bentuk data dan alur aplikasi.
