## 1. Apa itu HTTP?
HTTP (*Hypertext Transfer Protocol*) adalah "bahasa" atau aturan yang digunakan browser (Client) untuk berbicara dengan Server. Setiap kali Anda mengetik URL, browser mengirimkan **HTTP Request** dan server membalas dengan **HTTP Response**.

---

## 2. HTTP Methods (Jenis Permintaan)
Tidak semua permintaan itu sama. Ada beberapa cara utama untuk berinteraksi dengan server:
- **GET**: Meminta data (seperti membaca artikel).
- **POST**: Mengirim data baru (seperti mendaftar akun).
- **PUT/PATCH**: Memperbarui data yang sudah ada.
- **DELETE**: Menghapus data.

---

## 3. HTTP Status Codes (Kode Balasan)
Pernah melihat error **404 Not Found**? Itu adalah salah satu dari kode status HTTP. Berikut adalah kelompok kodenya:
- **2xx (Success)**: Semuanya lancar! (Contoh: `200 OK`).
- **3xx (Redirection)**: Alamat sudah pindah.
- **4xx (Client Error)**: Anda melakukan kesalahan. (Contoh: `404 Not Found`, `401 Unauthorized`).
- **5xx (Server Error)**: Server sedang bermasalah. (Contoh: `500 Internal Server Error`).

---

## 4. Struktur HTTP Request
Sebuah permintaan biasanya berisi:
1.  **URL**: Tujuan permintaan.
2.  **Header**: Informasi tambahan (seperti jenis browser atau token keamanan).
3.  **Body**: Data yang dikirim (biasanya dalam format JSON).

---

## 5. Analogi: Pesan Antar Makanan
- **URL**: Alamat restoran.
- **Method**: Apakah Anda ingin "Beli Makanan Baru" (POST) atau "Cek Menu" (GET)?
- **Status Code**: "Pesanan Diterima" (200), "Menu Habis" (404), atau "Dapur Kebakaran" (500).
