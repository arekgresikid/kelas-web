## DNS sebagai buku telepon internet
Jika internet adalah sebuah kota besar, maka DNS adalah buku teleponnya. Saat Anda mencari "google.com", DNS akan memberikan alamat IP server Google agar browser Anda tahu ke mana harus pergi.

## Cara kerja DNS lookup
Proses pencarian DNS melibatkan beberapa tahap:
1. **DNS Resolver**: ISP Anda menanyakan alamat IP.
2. **Root Server**: Memberitahu letak TLD server (misalnya `.com`).
3. **TLD Server**: Memberitahu letak Nameserver domain tersebut.
4. **Authoritative Nameserver**: Memberikan jawaban akhir berupa alamat IP.

## DNS caching
Untuk mempercepat akses, sistem operasi dan browser biasanya menyimpan hasil pencarian DNS untuk sementara waktu. Inilah yang disebut **Caching**. Jika Anda sering mengunjungi sebuah web, browser tidak perlu bertanya ke DNS server setiap kali.

## Contoh analogi sederhana
Bayangkan Anda ingin menelepon teman bernama "Andi". Anda tidak hafal nomornya, jadi Anda mencari di kontak HP. Nama "Andi" adalah Domain, dan nomor teleponnya adalah IP Address. Daftar kontak di HP Anda adalah DNS.
